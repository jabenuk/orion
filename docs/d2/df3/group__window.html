Detailed Description

Using this abstracted window system is not required, but it is recommended so that oriTerminate() can automatically free your window(s) and terminate GLFW. GLFW will be implicitly initialised on the first Orion window creation, but it can also be explicitly initialised with oriInitGLFW().

Any undocumented functions in this section are simply abstractions of GLFW functions. For documentation on those functions, see GLFW's window reference (linked below) and look for the function of the same name (just replace ori with glfw in the function name).

See also
GLFW window reference

To use the windowing abstractions ('orionwin'), include <orionwin.h>. None of this system will be included otherwise, so the main namespace won't be polluted if you want to manage windows in a different way.

◆ oriInitGLFW()

void oriInitGLFW ( )

This function is automatically called when the first oriWindow is created. Calling it after creating the first window won't do any harm; the function will just return early. The only real reason for this abstraction existing is for the ability to re-initialise GLFW without terminating Orion.

Note
The result of glfwInit() will automatically be checked and the program will halt (with a debug message) should GLFW fail to initialise.
See also
GLFW reference/glfwInit()
